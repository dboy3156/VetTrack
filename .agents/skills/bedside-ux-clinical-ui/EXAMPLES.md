# Bedside UX review examples

These examples contrast generic UI feedback with **clinical-grade** review tone and severity.

## Case 1: High-stress accessibility

- **Bad review**: "The button is 32px tall, which follows standard web patterns."
- **Clinical review**: "P1 risk: The action button is too small for bedside use where gloves or tremors might be present. Increase touch target to at least 48px to minimize error during ICU rounds."

## Case 2: Context switching and cognitive load

- **Bad review**: "Opening a modal here keeps the user on the same page."
- **Clinical review**: "P2 risk: Using a nested modal for medication confirmation obscures the patient's current vitals in the background. Recommend an inline slide-over or non-blocking notification to maintain context."

## Case 3: Visual hierarchy

- **Bad review**: "The font color is light gray to look modern."
- **Clinical review**: "P0 risk: Low-contrast text for dose amount fails readability under hospital lighting. Use high-contrast body text or bold weight for critical clinical values."
