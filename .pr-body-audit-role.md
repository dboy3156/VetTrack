## Summary
Adds **actor role** to audit log **metadata** (no DB migration). New writes include `metadata.actorRole` when callers pass `actorRole` on `logAudit()`, merged centrally so existing `metadata` keys are preserved.

## Server
- `logAudit` accepts optional `actorRole`; merged into JSONB metadata unless `metadata.actorRole` is already set.
- `resolveAuditActorRole(req)` returns `effectiveRole ?? authUser.role` (lowercase) for shift-aware auditing.
- Route/service coverage now includes: **equipment**, **users**, **folders**, **rooms**, **alert-acks**, **shifts** CSV import, **returns**, **procurement** (create/submit/receive/cancel), **appointments/tasks**, **medication tasks**, and task **automation/notification** events.
- **User sync / login** audit rows use the **subject user’s** DB role for `actorRole` (performed by self), not the HTTP session.

## UI
- Admin audit log list shows a compact **Role:** line when `metadata.actorRole` is present.
